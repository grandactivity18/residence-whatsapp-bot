# WhatsApp автоответчик для банкетного зала

Это небольшой бот для WhatsApp Business через Meta WhatsApp Cloud API.

Что умеет:

- отвечает клиенту меню с разделами;
- рассказывает про зал, меню, цены, бронь и частые вопросы;
- может отправлять меню картинкой по ссылке из `menuImageUrl`;
- переводит клиента на администратора;
- отвечает через ИИ на свободные вопросы клиента, если включен `AI_ENABLED`;
- хранит последние сообщения в `data/sessions.json`;
- дает простую админ-панель для ответа клиенту.

## 1. Заполнить информацию о зале

Откройте `banquet-info.json` и замените примерные данные на реальные:

- название зала;
- город и адрес;
- телефон;
- вместимость;
- пункты меню;
- цены;
- ссылку на фото меню в `menuImageUrl`, если меню нужно отправлять картинкой;
- условия брони;
- частые вопросы.

Для фото меню вставьте прямую публичную ссылку:

```json
"menuImageUrl": "https://example.com/menu.jpg"
```

Если оставить пусто:

```json
"menuImageUrl": ""
```

бот отправит только текстовое меню.

## 2. Настроить ключи

Скопируйте `.env.example` в `.env` и заполните:

```env
WHATSAPP_TOKEN=ваш_токен_Meta
PHONE_NUMBER_ID=id_номера_WhatsApp_Cloud_API
VERIFY_TOKEN=секретная_строка_для_webhook
ADMIN_PANEL_TOKEN=секрет_для_админ_панели
ADMIN_PHONE_NUMBER=77001234567
AI_ENABLED=true
OPENAI_API_KEY=ваш_ключ_ИИ
OPENAI_MODEL=gpt-4o-mini
OPENAI_BASE_URL=https://api.openai.com/v1
```

`ADMIN_PHONE_NUMBER` пишется без плюса, пробелов и скобок.

Если `AI_ENABLED=true`, бот будет отвечать на вопросы своими словами, но только на основе данных из `banquet-info.json`. Если клиент хочет бронь, расчет, проверку даты или разговор с человеком, бот переводит диалог администратору.

## 3. Запустить

```bash
npm start
```

Если PowerShell не запускает `npm`, используйте:

```bash
npm.cmd start
```

Локальный адрес:

```text
http://localhost:3000
```

Админ-панель:

```text
http://localhost:3000/admin?token=ВАШ_ADMIN_PANEL_TOKEN
```

## 4. Подключить WhatsApp

В настройках Meta WhatsApp Cloud API укажите webhook:

```text
https://ваш-домен/webhook
```

Verify token должен совпадать с `VERIFY_TOKEN` из `.env`.

Для локальной проверки можно временно открыть сервер наружу через ngrok или аналогичный туннель.

## Сценарий сообщений

Первое сообщение клиенту:

```text
Здравствуйте! Это Название банкетного зала, Ваш город.
Я помогу быстро узнать основную информацию.

Выберите раздел цифрой:
1. Банкетный зал
2. Меню
3. Цены
4. Проверить дату
5. Связаться с администратором
6. Частые вопросы
```

Если клиент выбирает `5` или пишет "администратор", бот переводит диалог в ручной режим и больше не перебивает клиента автоматическими ответами.
